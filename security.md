# Security Policy for dai-wallet-app

Thank you for taking the time to responsibly disclose security issues affecting dai-wallet-app. We take security seriously and appreciate coordinated, responsible disclosure.

## Supported Versions

We support only the latest stable release on the default branch. If you are reporting an issue in an older release, please state the exact tag/commit and branch.

## Reporting a Vulnerability

Preferred options to report a vulnerability:

1. GitHub Security Advisories: If this repository exposes the "Security" tab for private disclosures, please use a private security advisory and add the repository owners.
2. Email: If you can't use the Security Advisory workflow, send an email to: security@danishahmed111.dev

When reporting, please include:
- A clear description of the vulnerability and the potential impact.
- Steps to reproduce, with minimal reproduction steps or a small test case if possible.
- Affected versions, branch and commit SHA if known.
- Any logs, screenshots, or code excerpts needed to reproduce.
- A suggested mitigation or workaround if you have one.

We will:
- Acknowledge receipt within 72 hours.
- Triage and confirm the issue, and provide an estimated timeline for a fix.
- Coordinate disclosure with the reporter. We aim to publish fixes and public advisories only after a patch is available (typically within 90 days), or sooner if the reporter requests an expedited disclosure.

## PGP / Secure Communication

If you prefer encrypted email, request our PGP key by sending an initial unencrypted email to the address above describing who you are and the file(s) you will send encrypted. We will reply with a public key within 72 hours.

## What We Ask of Reporters

- Do not publicly disclose the vulnerability before a fix is available or we agree on a disclosure timeline.
- Avoid large-scale data collection or exploitation when validating issues.
- Provide reasonable time for maintainers to create and publish a patch.

## Remediation and CVE

If the issue qualifies for a CVE, we will coordinate assignment either directly or via the reporter. We will credit researchers who follow this policy when the issue is publicly disclosed, unless the reporter requests anonymity.

## Sensitive Data

Please do not include plaintext passwords, private keys, or other sensitive credentials in your report. If necessary to reproduce, provide test credentials with clearly limited scope and expiry.
